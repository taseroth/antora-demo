Welcome to Project <Demo>

This is an example project software (architecture) documentation for project setups that I often encounter at my customers.

The aim is to set up a documentation structure for re-use. It follows the doc-as-code paradigm and is based on Arc42 templates, uses Acsiidoc to write the documentation in and PlantUML to render images.

As a consultant for Neo4j these projects are centered around a graph database, and I will adjust the Arc42 templates to the needs of those projects.

The example application is for a 'Bill Of Materials' use-case, but no actual software artefacts are included as this is all about documentation.

Those solutions typically consist of a few building blocks (see 03_system_scope_and_context.adoc) that each live in their own git repository. While it is possible to document each such (sub)project using tools such as the excellent docToolchain, combining them together requires a different approach.

Antora provides exactly this feature in allowing to provide multiple git repositories as source and combining the output into a single document.

This repository provides the configuration via the antora-playbook.yml configuration file as well as hosting the documentation that is common for all subproject (architecture overview, ADRs, stakeholders,…​)

1. Usage

  • Make sure you have a current LTS version of Node installed

  • Install Antora,

  • Clone this repository: git clone https://github.com/taseroth/antora-demo.git

  • run: npx antora antora-playbook.yml .

There is also an Antora playbook for local usage to test changes without committing them first. You can run npx antora antora-playbook-local.yml to use the files checkout locally.
